# Delivery Route Optimisation вЂ” Final Year Project

A full-stack delivery route optimisation system built for a small-scale delivery prototype. Drivers scan delivery barcodes, the system optimises the route using a trained ML model combined with classical heuristics, and the result is exported to Google Maps for navigation.

---

## Overview

The system combines two route optimisation approaches:

- **Nearest Neighbour + 2-opt** вЂ” a classical greedy heuristic refined by local search
- **XGBoost ML model + 2-opt** вЂ” a trained classifier that learns to imitate the NN+2-opt teacher route, then is further refined with 2-opt

At runtime, both algorithms generate a route and the shorter one is selected automatically (`auto` mode). The ML model alone outperforms standard Nearest Neighbour, and ML + 2-opt is competitive with NN + 2-opt on unseen problems.

## Setup

### Requirements

- Python 3.12+
- PostgreSQL
- Node not required (vanilla JS frontend)

### Install dependencies

```bash
pip install -r requirements.txt
```

### Environment variables

Create a `.env` file in the project root:

```
DATABASE_URL=postgresql://user:password@localhost:5432/yourdb
```

### Set up the database

```bash
psql -U youruser -d yourdb -f api/db/schema.sql
python -m api.db.seed
```

---

## Training the ML Model

### 1. Generate training datasets

```bash
python -m data.generate_training_datasets --count 2000 --min-n 5 --max-n 30 --wipe
```

Datasets use seeds `0вЂ“999`. Evaluation uses seeds `1000+` to ensure no data leakage.

### 2. Train the model

```bash
python -m model.train
```

Trains an XGBoost classifier on teacher routes generated by Nearest Neighbour + 2-opt. Saves the model to `model/trained_route_model.joblib`.

---

## Running the API

```bash
python -m api.app
```

API runs at `http://127.0.0.1:5000`.

### Key endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/api/deliveries` | All deliveries |
| GET | `/api/deliveries/pending` | Pending deliveries only |
| GET | `/api/deliveries/barcode/<barcode>` | Lookup by barcode |
| GET | `/api/employees/<number>` | Employee login |
| POST | `/api/deliveries/optimise` | Optimise a route |

#### Optimise endpoint

```json
POST /api/deliveries/optimise
{
  "deliveries": [...],
  "method": "auto"
}
```

Supported methods: `ml`, `ml_two_opt`, `nn_two_opt`, `auto`. `auto` runs both ML+2opt and NN+2opt and returns the shorter route.

## Notes

- The distance matrix uses the **Haversine formula** for great-circle distances, accounting for the curvature of the Earth.
- Google Maps is used for **navigation only** вЂ” it drops pins at the optimised stop order. The routing algorithm is entirely this system's own.
- The ML model is intentionally lightweight вЂ” XGBoost with engineered features rather than a neural network, keeping inference fast and resource requirements low.
- Random Forest was trialled first but XGBoost produced better results, particularly on class 1 recall which is the meaningful metric given the structural class imbalance of the training data.
