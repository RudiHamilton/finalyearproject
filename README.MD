# Delivery Route Optimisation — Final Year Project

A full-stack delivery route optimisation system built for a small-scale delivery prototype. Drivers scan delivery barcodes, the system optimises the route using a trained ML model combined with classical heuristics, and the result is exported to Google Maps for navigation.

---

## Overview

The system combines two route optimisation approaches:

- **Nearest Neighbour + 2-opt** — a classical greedy heuristic refined by local search
- **XGBoost ML model + 2-opt** — a trained classifier that learns to imitate the NN+2-opt teacher route, then is further refined with 2-opt

At runtime, both algorithms generate a route and the shorter one is selected automatically (`auto` mode). The ML model alone outperforms standard Nearest Neighbour, and ML + 2-opt is competitive with NN + 2-opt on unseen problems.

## Setup

### Requirements

- Python 3.12+
- PostgreSQL
- Node not required (vanilla JS frontend)

### Install dependencies

```bash
pip install -r requirements.txt
```

### Environment variables

Create a `.env` file in the project root:

```
DATABASE_URL=postgresql://user:password@localhost:5432/yourdb
```

### Set up the database

```bash
psql -U youruser -d yourdb -f api/db/schema.sql
python -m api.db.seed
```

---

## Training the ML Model

### 1. Generate training datasets

```bash
python -m data.generate_training_datasets --count 2000 --min-n 5 --max-n 30 --wipe
```

Datasets use seeds `0–999`. Evaluation uses seeds `1000+` to ensure no data leakage.

### 2. Train the model

```bash
python -m model.train
```

Trains an XGBoost classifier on teacher routes generated by Nearest Neighbour + 2-opt. Saves the model to `model/trained_route_model.joblib`.

---

Start the application:

```bash
python -m api.app
```

Open `http://127.0.0.1:8000` in your browser. Flask serves both the frontend and the API — no separate server required. The frontend handles all communication with the API automatically.

---

## Generating Barcodes

```bash
python -m data.generate_barcode_page
```

Generates barcode images and a printable HTML label page at `data/generated_barcodes/barcode_labels.html`. Barcodes are Code128 and can be scanned with a physical barcode scanner.

---

## Running Experiments

```bash
python -m experiments.compare_algorithms
```

Benchmarks all algorithms across 20 unseen seeds and saves results to `experiments/results.json`.

```bash
python -m experiments.export_to_csv
```

Converts all JSON experiment results to CSV for reporting.

## Notes

- The distance matrix uses the **Haversine formula** for great-circle distances, accounting for the curvature of the Earth.
- Google Maps is used for **navigation only** — it drops pins at the optimised stop order. The routing algorithm is entirely this system's own.
- The ML model is intentionally lightweight — XGBoost with engineered features rather than a neural network, keeping inference fast and resource requirements low.
- Random Forest was trialled first but XGBoost produced better results, particularly on class 1 recall which is the meaningful metric given the structural class imbalance of the training data.
